27:T6a1f,<p>These notes cover the core concepts of the Information Bottleneck method, widely used in machine learning and theoretical neuroscience.
We start by defining the fundamental tension in learning and representation.</p>
<h4 id="learning-design-goals">Learning Design Goals<a class="anchor" aria-hidden="true" hidden href="#learning-design-goals">#</a></h4>
<p><strong>Compression</strong>: The representation should be as simple as possible (Occam's Razor).
<strong>Relevance</strong>: The representation must retain enough information to predict the target variable accurately.
<strong>Generalization</strong>: The system should perform well on unseen data, which is often a function of finding the right balance between the first two.
<strong>Invariance</strong>: The representation should ignore "nuisance variables" (noise) present in the input that do not correlate with the output.</p>
<blockquote>
<p>“The goal of deep learning is to find a representation of the input that is maximally informative about the output, while being maximally compressed regarding the input.” ~<em>Naftali Tishby</em></p>
</blockquote>
<p>This node is strictly related to Rate-Distorsion Theory, but we apply it here specifically for supervised learning tasks where <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span> is a specific target, not just a reconstruction of <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span>.</p>
<h3 id="the-core-mechanism">The Core Mechanism<a class="anchor" aria-hidden="true" hidden href="#the-core-mechanism">#</a></h3>
<h4 id="the-markov-chain-setup">The Markov Chain Setup<a class="anchor" aria-hidden="true" hidden href="#the-markov-chain-setup">#</a></h4>
<p>See <a href="/notes/markov-chains/">Markov Chains</a>.
We assume a standard supervised learning setting. We have an input variable <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span> (e.g., an image) and a target variable <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span> (e.g., a label). We want to learn an intermediate representation <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> (e.g., a hidden layer in a neural network).
The fundamental structural assumption is the Markov Chain:</p>
<p><span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">→</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">→</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span></p>
<p>This implies that <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> depends <em>only</em> on <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span>. <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> cannot see <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span> directly; it can only learn about <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span> through <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span>.</p>
<h4 id="mutual-information-recall">Mutual Information Recall<a class="anchor" aria-hidden="true" hidden href="#mutual-information-recall">#</a></h4>
<p>To understand the bottleneck, we use <strong>Mutual Information</strong> (<span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span></span></span></span>).
<span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mclose">)</span></span></span></span> measures how much information <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> contains about <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span>.
<span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span><span class="mclose">)</span></span></span></span> measures how much information <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> preserves about <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span>.
If <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mclose">)</span></span></span></span> is high, we are "memorizing" the input (high complexity).
If <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span><span class="mclose">)</span></span></span></span> is high, we are "predicting" the output (high accuracy).</p>
<h3 id="the-optimization-problem">The Optimization Problem<a class="anchor" aria-hidden="true" hidden href="#the-optimization-problem">#</a></h3>
<h4 id="the-lagrangian-formulation">The Lagrangian Formulation<a class="anchor" aria-hidden="true" hidden href="#the-lagrangian-formulation">#</a></h4>
<blockquote>
<p>The Information Bottleneck method seeks a representation <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> that minimizes the information about the input <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span> while maximizing the information about the output <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span>.</p>
</blockquote>
<p>We formulate this as a minimization problem of a Lagrangian function <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathcal">L</span></span></span></span>:</p>
<p><span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1.0385em;vertical-align:-0.3552em;"></span><span class="mop"><span class="mop">min</span><span class="msupsub"><span class="vlist-t vlist-t2"><span class="vlist-r"><span class="vlist" style="height:0.3448em;"><span style="top:-2.5198em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mathnormal mtight">p</span><span class="mopen mtight">(</span><span class="mord mathnormal mtight">t</span><span class="mord mtight">∣</span><span class="mord mathnormal mtight">x</span><span class="mclose mtight">)</span></span></span></span></span><span class="vlist-s">​</span></span><span class="vlist-r"><span class="vlist" style="height:0.3552em;"><span></span></span></span></span></span></span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathcal">L</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">=</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mclose">)</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">−</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span><span class="mclose">)</span></span></span></span>
Where <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span></span></span></span> is a Lagrange multiplier that controls the trade-off.</p>
<ul>
<li><strong>Minimizing <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mclose">)</span></span></span></span></strong>: This pushes the representation to "forget" the input details (Compression).</li>
<li><strong>Maximizing <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span><span class="mclose">)</span></span></span></span></strong>: This forces the representation to keep what matters for the target (Prediction).</li>
</ul>
<p>As we vary <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span></span></span></span>, we trace out an optimal curve known as the <strong>IB Curve</strong>.</p>
<h4 id="the-role-of-beta">The Role of Beta<a class="anchor" aria-hidden="true" hidden href="#the-role-of-beta">#</a></h4>
<p><span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span></span></span></span> acts as a "focus" parameter.</p>
<ul>
<li><strong>Low <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span></span></span></span></strong>: We prioritize compression. <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> becomes very simple, effectively a coarse clustering of <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span>. We lose details about <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span>.</li>
<li><strong>High <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span></span></span></span></strong>: We prioritize prediction. <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span></span></span></span> becomes complex and retains more details from <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span> to ensure <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span> is captured perfectly.</li>
<li><strong><span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0528em;">β</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">→</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.4306em;"></span><span class="mord">∞</span></span></span></span></strong>: This approaches the "Sufficient Statistics" limit, where we keep <em>everything</em> relevant to <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span>, regardless of cost.</li>
</ul>
<h3 id="application-to-deep-learning">Application to Deep Learning<a class="anchor" aria-hidden="true" hidden href="#application-to-deep-learning">#</a></h3>
<h4 id="tishbys-theory-of-dnns">Tishby’s Theory of DNNs<a class="anchor" aria-hidden="true" hidden href="#tishbys-theory-of-dnns">#</a></h4>
<p>This is the controversial but fascinating part. Tishby proposed that Deep Neural Networks (DNNs) naturally optimize the IB bound layer-by-layer.
We can visualize the training process of a network in the <strong>Information Plane</strong> (x-axis: <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mclose">)</span></span></span></span>, y-axis: <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span><span class="mclose">)</span></span></span></span>).</p>























<table><thead><tr><th><strong>Phase</strong></th><th><strong>Description</strong></th><th><strong>What happens to I(X;T)?</strong></th><th><strong>What happens to I(T;Y)?</strong></th></tr></thead><tbody><tr><td><strong>Fitting Phase</strong></td><td>Early training. The network learns to label the data.</td><td><strong>Increases rapidly</strong>. The network absorbs information from input.</td><td><strong>Increases</strong>. The network learns to predict the label.</td></tr><tr><td><strong>Compression Phase</strong></td><td>Later training. SGD diffusion noise kicks in.</td><td><strong>Decreases</strong>. The network "forgets" irrelevant input noise.</td><td><strong>Stays high/Plateaus</strong>. Prediction accuracy is maintained.</td></tr></tbody></table>
<p>The claim is that <strong>Generalization</strong> (doing well on test data) is a direct result of this compression phase. By forgetting the specifics of the training set (lowering <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">I</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span><span class="mpunct">;</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.1389em;">T</span><span class="mclose">)</span></span></span></span>), the network avoids overfitting.</p>
<h4 id="geometric-interpretation">Geometric Interpretation<a class="anchor" aria-hidden="true" hidden href="#geometric-interpretation">#</a></h4>
<p>Think of the input space as a manifold.</p>
<ul>
<li>The <strong>Fitting</strong> phase is like stretching the manifold to separate the classes.</li>
<li>The <strong>Compression</strong> phase is like collapsing the dimensions of the manifold that don't help with separation.</li>
</ul>
<h4 id="criticisms-and-nuance">Criticisms and nuance<a class="anchor" aria-hidden="true" hidden href="#criticisms-and-nuance">#</a></h4>
<p>While the theory is elegant, it has faced criticism (e.g., Saxe et al.).</p>
<ol>
<li><strong>ReLU Networks</strong>: Some argue that networks with ReLU activation don't strictly compress mutual information in the same way (due to binning issues/infinite values).</li>
<li><strong>Invariance vs Compression</strong>: Is the network actually "forgetting" data, or just becoming invariant to transformations?</li>
</ol>
<h3 id="comparison-with-other-principles">Comparison with other Principles<a class="anchor" aria-hidden="true" hidden href="#comparison-with-other-principles">#</a></h3>

<h4 id="variational-information-bottleneck-vib">Variational Information Bottleneck (VIB)<a class="anchor" aria-hidden="true" hidden href="#variational-information-bottleneck-vib">#</a></h4>
<p>Since calculating Mutual Information is computationally hard in high dimensions (like pixel space), we often use a <strong>variational approximation</strong>.
This is very similar to how <a href="/notes/autoencoders/#variational-autoencoders">Autoencoders#Variational Autoencoders</a> (VAEs) work, but with a specific loss function designed to satisfy the IB principle.</p>
<pre><code class="language-python"># Pseudo-code for a VIB Loss function
# beta is the tradeoff parameter
def vib_loss(y_true, y_pred, z_mean, z_log_var, beta):
    # 1. Prediction Loss (Cross Entropy) -> Maximize I(T;Y)
    prediction_loss = cross_entropy(y_true, y_pred)
    
    # 2. Compression Loss (KL Divergence) -> Minimize I(X;T)
    # We want T (z) to be close to a standard prior (forgetting X)
    kl_loss = -0.5 * sum(1 + z_log_var - square(z_mean) - exp(z_log_var))
    
    return prediction_loss + beta * kl_loss
</code></pre>
<h4 id="summary">Summary<a class="anchor" aria-hidden="true" hidden href="#summary">#</a></h4>
<p>The Information Bottleneck is a principled framework for understanding <strong>what</strong> a model learns.
It suggests that "learning is forgetting": to generalize well, you must filter out the noise (irrelevant bits of <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0785em;">X</span></span></span></span>) and keep only the signal (bits relevant to <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.2222em;">Y</span></span></span></span>).</p>
<blockquote>
<p>"You cannot learn what is relevant without learning what to ignore."</p>
