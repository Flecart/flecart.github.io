22:Tbd7a,<h2 id="introduzione">Introduzione<a class="anchor" aria-hidden="true" hidden href="#introduzione">#</a></h2>
<h3 id="limportanza-del-topic">L’importanza del topic<a class="anchor" aria-hidden="true" hidden href="#limportanza-del-topic">#</a></h3>
<p>Gli algoritmi di ordinamento sono molto di base per la comprensione dell'ampio raggio degli algoritmi. Utilizzano l'analisi, introducono tecniche di risoluzione dei problemi computazionali come greedy, divide et impera e simile. Permettono un primo uso di astrazioni e l'analisi di sottoproblemi.</p>
<h3 id="il-problema">Il problema<a class="anchor" aria-hidden="true" hidden href="#il-problema">#</a></h3>
<p>Il problema è trovare una permutazione di un insieme di numeri iniziali tale per cui tale insieme di numeri si ordinato:</p>
<p>Questo si può fare con qualunque collezione confrontabile fra di loro.</p>
<ul>
<li>
<p>Slide</p>
<p><img src="/images/notes/image/universita/ex-notion/Algoritmi di ordinamento/Untitled.png" style="width: 100%" class="center" alt="Untitled"></p>
</li>
</ul>
<p><strong>Loco:</strong> non vengono utilizzati array di appoggio, per esempio un quicksort sarà di in loco, mentre mergesort no.</p>
<p><strong>Stabile:</strong> se due elementi hanno la stessa chiave, questi compaiono con lo stesso ordine nell'array ordinato, per esempio radix sort è stabile, merge sort è stabile.</p>
<h2 id="algoritmi-incrementali">Algoritmi incrementali<a class="anchor" aria-hidden="true" hidden href="#algoritmi-incrementali">#</a></h2>
<p>Si possono dire incrementali algoritmi che si dimostrano in "maniera greedy", ossia creano un sottoaray ordinato k, e al prossimo passo provano a creare un array k + 1.</p>
<h3 id="selection-sort">Selection sort<a class="anchor" aria-hidden="true" hidden href="#selection-sort">#</a></h3>
<p>Si trova l'elemento più piccolo all'interno di <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0315em;">k</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">+</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord">1</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal">n</span><span class="mclose">)</span></span></span></span> e lo si swappa con quello in posizione k + 1, si continua così fino a quando l'array non sia ordinato. <strong>NO stabile</strong> (spara dall'altra parte l'elemento con cui swappa).</p>
<p>Una proprietà che possiede selection ma non insertion è che:</p>
<p><span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.7335em;vertical-align:-0.0391em;"></span><span class="mord">∀</span><span class="mord mathnormal">e</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">∈</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mopen">(</span><span class="mord">1</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.0315em;">k</span><span class="mclose">)</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord">∀</span><span class="mord mathnormal">b</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">∈</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0315em;">k</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">+</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord">1</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal">n</span><span class="mclose">)</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal">e</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">&#x3C;</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6944em;"></span><span class="mord mathnormal">b</span></span></span></span> ma con insertion non vale, questo permette di concludere il ciclo un elemento prima, mentre per insertion bisogna andare fino alla fine.</p>
<ul>
<li>
<p>Codice java</p>
<pre><code class="language-java">public static void selectionSort(Comparable A[]) {
	for (int k = 0; k &#x3C; A.length - 1; k++) {
		// cerca il minimo A[m] in A[k..n-1]
		int m = k;
		for (int j = k + 1; j &#x3C; A.length; j++)
			if (A[j].compareTo(A[m]) &#x3C; 0) m = j;
		// scambia A[k] con A[m]
		if (m != k) {
			Comparable temp = A[m];
			A[m] = A[k];
			A[k] = temp;
		}
	}
}
</code></pre>
</li>
<li>
<p>Analisi del costo</p>
<p><img src="/images/notes/image/universita/ex-notion/Algoritmi di ordinamento/Untitled 1.png" style="width: 100%" class="center" alt="Untitled 1"></p>
</li>
</ul>
<h3 id="insertion-sort">Insertion sort<a class="anchor" aria-hidden="true" hidden href="#insertion-sort">#</a></h3>
<p>Si inserisce l'elemento k + 1 nella posizione corretta in <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mopen">(</span><span class="mord">1</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.0315em;">k</span><span class="mclose">)</span></span></span></span>. <strong>è stabile</strong></p>
<ul>
<li>
<p>Codice java</p>
<pre><code class="language-java">public static void insertionSort(Comparable A[]) {
	for (int k = 1; k &#x3C;= A.length - 1; k++) {
		int j;
		Comparable x = A[k];
		// cerca la posizione j in cui inserire A[k]
		for (j = 0; j &#x3C; k; j++) {
			if (A[j].compareTo(x) > 0) break;
		}
		if (j &#x3C; k) {
			// Sposta A[j..k-1] in A[j+1..k]
			for (int t = k; t > j; t--)
				A[t] = A[t – 1];
			// Inserisci A[k] in posizione j
			A[j] = x;
		}
	}
}
</code></pre>
</li>

<p>Si fa "galleggiare" l'elemento massimo fino alla fine, creando un array ordinato per <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0315em;">k</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">+</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord">1</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal">n</span><span class="mclose">)</span></span></span></span>, inoltre si ha che ogni elemento in <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mopen">(</span><span class="mord">1</span><span class="mpunct">,</span><span class="mspace" style="margin-right:0.1667em;"></span><span class="mord mathnormal" style="margin-right:0.0315em;">k</span><span class="mclose">)</span></span></span></span> è minore del primo insieme.</p>

		boolean scambiAvvenuti = false;
		for (int j = 1; j &#x3C;= A.length - i; j++) {

				Comparable temp = A[j - 1];
				A[j - 1] = A[j];
				A[j] = temp;
				scambiAvvenuti = true;

}

<h2 id="algoritmi-divide-et-impera">Algoritmi Divide et Impera<a class="anchor" aria-hidden="true" hidden href="#algoritmi-divide-et-impera">#</a></h2>
<h3 id="quicksort">Quicksort<a class="anchor" aria-hidden="true" hidden href="#quicksort">#</a></h3>
<p><strong>Storia e dell'algoritmo</strong></p>
<p>Inventato da <a href="https://en.wikipedia.org/wiki/Tony_Hoare">Hoare</a> nel 1962, cerca di creare ordinare l'array secondo il metodo di divide et impera che si divide in 3 passi.</p>
<ol>
<li>Dividere il problema in sottoproblemi più semplici</li>
<li>Risolvere i sottoproblemi individualmente</li>
<li>Mettere assieme le soluzioni parziali</li>
</ol>
<p><strong>Introduzione all'algoritmo</strong></p>
<p>Si prende un valore della sequenza da ordinare, che chiamiamo <strong>pivot</strong>. Vogliamo creare così 2 array (in loco) tali che una sia sempre più piccola di questo valore, l'altra sempre più grande, mettiamo il pivot in mezzo e andiamo a ordinare questo array.</p>
<ul>
<li>
<p>Codice Java</p>
<pre><code class="language-java">public static void quickSortRec(Comparable A[], int i, int f) {
	if (i >= f) return;
	int m = partition(A, i, f);
	quickSortRec(A, i, m - 1);
	quickSortRec(A, m+1, f);
}
</code></pre>
<pre><code class="language-java">private static int partition(Comparable A[], int i, int f) {
	int inf = i, sup = f + 1;
	Comparable temp, x = A[i];
	while (true) {
		do {
			inf++;
		} while (inf &#x3C;= f &#x26;&#x26; A[inf].compareTo(x) &#x3C;= 0);
		do {
			sup--;
		} while (A[sup].compareTo(x) > 0);
		if (inf &#x3C; sup) {
			temp = A[inf];
			A[inf] = A[sup];
			A[sup] = temp;
		} else break;
	}
	temp = A[i];
	A[i] = A[sup];
	A[sup] = temp;
	return sup;
}
</code></pre>
</li>
</ul>

<h3 id="mergesort">MergeSort<a class="anchor" aria-hidden="true" hidden href="#mergesort">#</a></h3>

<li>

<pre><code class="language-java">public static void mergeSort(Comparable A[]) {
	mergeSortRec(A, 0, A.length - 1);

private static void mergeSortRec(Comparable A[], int i, int f) {
	if (i >= f) return;
	int m = (i + f) / 2;
	mergeSortRec(A, i, m);
	mergeSortRec(A, m + 1, f);
	merge(A, i, m, f);

</code></pre>
<pre><code class="language-java">private static void merge(Comparable A[], int i1, int f1, int f2)
{
	Comparable[] X = new Comparable[f2 - i1 + 1];
	int i = 0, i2 = f1 + 1, k = i1;
	while (i1 &#x3C;= f1 &#x26;&#x26; i2 &#x3C;= f2) {
	if (A[i1].compareTo(A[i2]) &#x3C; 0)
		X[i++] = A[i1++];
	else
		X[i++] = A[i2++];
	}
	if (i1 &#x3C;= f1)
		for (int j = i1; j &#x3C;= f1; j++, i++) X[i] = A[j];
	else
		for (int j = i2; j &#x3C;= f2; j++, i++) X[i] = A[j];
	for (int t = 0; k &#x3C;= f2; k++, t++) A[k] = X[t];
}
</code></pre>

<h2 id="heap">Heap<a class="anchor" aria-hidden="true" hidden href="#heap">#</a></h2>
<p>Questo algoritmo per il sorting non utilizza il divide et impera, ma una struttura di dati utile, per questo motivo lo mettiamo in una sezione apparte, che abbia inserimento e rimozione in log n.</p>
<p>L'albero binario pone vincoli molto più forti perché l'intero sottoalbero destro e sinistro del nodo devono soddisfare certe proprietà</p>
<h3 id="albero-heap">Albero Heap<a class="anchor" aria-hidden="true" hidden href="#albero-heap">#</a></h3>
<p>Usiamo alberi completi, ovvero <strong>ogni noto ha grado 2</strong> tranne le foglie.</p>
<p>Possiamo anche avere un albero "quasi" completo, compattato a sinistra, ovvero le foglie possono avere buchi solamente a sinistra</p>
<p>Questo sarà l'albero binario utilizzato dalla heap.</p>
<h3 id="funzioni-per-la-heap-5">Funzioni per la heap (5)<a class="anchor" aria-hidden="true" hidden href="#funzioni-per-la-heap-5">#</a></h3>
<h4 id="heapify"><strong>Heapify</strong><a class="anchor" aria-hidden="true" hidden href="#heapify">#</a></h4>
<p>si cicla da n/2 + 1 fino al 1 (root)</p>
<p>per ogni elemento chiamo fixheap(i)</p>
<p>crea la heap in tempo lineare, a differenza dell'insertion in n volte</p>
<ul>
<li>
<p>Codice java</p>
<pre><code class="language-java">private static void heapify(Comparable S[], int n, int i) {
	if (i > n) return;
	heapify(S, n, 2 * i); // crea heap radicato in S[2*i]
	heapify(S, n, 2 * i + 1); // crea heap radicato in S[2*i+1]
	fixHeap(S, n, i);
}
// per trasformare un array S in uno heap:
// heapify(S, S.length, 1 );
</code></pre>
</li>
</ul>
<h4 id="fixheap-index"><strong>fixheap (index)</strong><a class="anchor" aria-hidden="true" hidden href="#fixheap-index">#</a></h4>
<p>Si prende i lminimo fra index, left(index) e right(index), se questo minimo è diverso da index, si swappa e si continua ricorsivamente finché o non cambia o sono una foglia.</p>
<ul>
<li>
<p>Codice java</p>
<pre><code class="language-java">private static void fixHeap(Comparable S[], int c, int i) {
	// c è la size attuale della heap
	int max = 2 * i; // figlio sinistro
	if (2 * i > c) return;
	if (2 * i + 1 &#x3C;= c &#x26;&#x26; S[2 * i].compareTo(S[2 * i + 1]) &#x3C; 0)
		max = 2 * i + 1; // figlio destro
	if (S[i].compareTo(S[max]) &#x3C; 0) {
		Comparable temp = S[max];
		S[max] = S[i];
		S[i] = temp;
		fixHeap(S, c, max);
	}
}
</code></pre>
</li>
</ul>
<p><strong>DeleteMax ()</strong></p>
<ol>
<li>Si scambia il massimo con l'ultimo</li>
<li>Si decrementa la size della heap di 1</li>
<li>Si chiama fix heap sul root</li>
</ol>
<ul>
<li>
<p>Codice Java mio</p>
<pre><code class="language-java">private static void deleteMax(Comparable S[], int c) {
	Comparable tmp = S[c];
	S[c] = S[1];
	S[1] = tmp;
	fixHeap(S, c - 1, 1);
}
</code></pre>
</li>
</ul>
<p><strong>Upper fix(index)</strong></p>
<p>È praticamente fixHeap, però va verso l'alto</p>
<p><strong>Insert(i)</strong></p>
<p>Metto l'elemento in fondo, aggiungo 1 alla size della heap e chiamo upper fix su questo elemento.</p>
<h3 id="heapsort">Heapsort<a class="anchor" aria-hidden="true" hidden href="#heapsort">#</a></h3>
<p>Finalmente avendo (almeno 3) le funzioni precedenti, possiamo andare a costruire una funzione di sorting in loco, l'heapsort</p>
<ul>
<li>
<p>Codice Java</p>
<pre><code class="language-java">public static void heapSort(Comparable S[]) {
	heapify(S, S.length - 1, 1);
	for (int c = (S.length - 1); c > 0; c--) {
		Comparable k = findMax(S);
		deleteMax(S, c);
		S[c] = k;
	}
}
</code></pre>
</li>
</ul>
<p>Quindi heapify O(n), mentre delete max n volte ho un n log n, quindi la complessità è di O(nlgn).</p>

<h2 id="ordinamento-lineare">Ordinamento lineare<a class="anchor" aria-hidden="true" hidden href="#ordinamento-lineare">#</a></h2>
<h3 id="counting-sort">Counting sort<a class="anchor" aria-hidden="true" hidden href="#counting-sort">#</a></h3>
<p>Vogliamo contare il numero di elementi in <strong>un range ben specificato</strong></p>
<p>L'ordinamento stabile non è ben specificato qui.</p>
<ul>
<li>
<p>Codice java</p>
<pre><code class="language-java">public static void countingSort(int[] A, int k) {
	int[] Y = new int[k];
	int j = 0;
	for (int i = 0; i &#x3C; k; i++) Y[i] = 0;
	for (int i = 0; i &#x3C; A.length; i++) Y[A[i]]++;
	for (int i = 0; i &#x3C; k; i++) {
		while (Y[i] > 0) {
			A[j] = i;
			j++;
			Y[i]--;
		}
	}
}
</code></pre>
</li>
<li>
<p>Analisi del costo</p>
<p>Sto facendo 1 ciclo su k.</p>
<p>1 ciclo su n</p>
<p>1 ciclo su k con al massimo n iterazioni. n + k</p>
<p>Quindi è in theta (n + k).</p>
</li>
</ul>
<p>Una cosa brutta di questo è che <strong>vale solo per INTERI.</strong></p>
<h3 id="bucket-sort">Bucket Sort<a class="anchor" aria-hidden="true" hidden href="#bucket-sort">#</a></h3>
<p>Bucket ha il vantaggio di essere eseguibile per ogni tipologia quindi anche i records!</p>
<p>Praticamente è uguale a counting sort, ma si utilzza array di liste!</p>
<ul>
<li>
<p>Pseudocodice</p>
<pre><code class="language-java">Algoritmo bucketSort(array X[1..n], intero k)
Sia Y un array di dimensione k
for i := 1 to k do
	Y[i]:=lista vuota
endfor
for i := 1 to n do
	Appendi X[i] alla lista Y[chiave(X[i])];
endfor
for i := 1 to k do
	copia ordinatamente in X gli elementi di Y[i]
endfor
</code></pre>
</li>
</ul>
<h3 id="radix-sort">Radix Sort<a class="anchor" aria-hidden="true" hidden href="#radix-sort">#</a></h3>
<p>Il problema dei precedenti è che c'è bisogno di molta memoria per rappresentare la nostra chiave, per questo motivo radix sort interviene per cercare di risolvere questa situazione sfavorevole per i precedenti.

<p>L'algoritmo è utilizzare un ordinamento stabile per fare l'ordinamento che ci serve.</p>
