29:T1344f,<p>Transformers, introduced in NLP language translation in <a href="http://arxiv.org/abs/1706.03762">(Vaswani et al. 2017)</a>, are one of the cornerstones of modern deep learning. For this reason, it is quite important to understand how they are done.</p>
<h3 id="introduction-to-transformers">Introduction to Transformers<a class="anchor" aria-hidden="true" hidden href="#introduction-to-transformers">#</a></h3>
<p>Transformers are called in this manner because they <em>transform</em> the input data space into another with the <em>same dimensionality</em>.</p>
<blockquote>
<p>The goal of the transformation is that the new space will have a richer internal representation that is better suited to solving downstream tasks. <a href="https://link.springer.com/10.1007/978-3-031-45468-4">(Bishop &#x26; Bishop 2024)</a></p>
</blockquote>
<h4 id="general-characteristics">General characteristics<a class="anchor" aria-hidden="true" hidden href="#general-characteristics">#</a></h4>
<blockquote>
<p>One major advantage of transformers is that <strong>transfer learning</strong> is very effective, so
that a transformer model can be trained on a large body of data and then the trained
model can be applied to many downstream tasks using some form of fine-tuning.</p>
</blockquote>
<blockquote>
<p>Moreover, the transformer is
especially well suited to massively parallel processing hardware such as graphical
processing units, or GPUs, allowing exceptionally large neural network language
models having of the order of a trillion (1012 ) parameters to be trained in reason-
able time</p>
</blockquote>
<blockquote>
<p>A powerful property of transformers is that we do not have to design a
new neural network architecture to handle a mix of different data types but instead
can simply <em>combine</em> the data variables into a joint set of tokens.</p>
</blockquote>
<h3 id="introduction-to-the-structure">Introduction to the structure<a class="anchor" aria-hidden="true" hidden href="#introduction-to-the-structure">#</a></h3>
<p>Transformers are just repeated blocks of attention layers, norms, MLP, followed by a final softmax on the final MLP layer, and preceded by a encoding layer.
The first encoding layer has to embed some information about the original structure:</p>
<ol>
<li>Semantic information about the input</li>
<li>Positional information about the input.
Then we use the transformer blocks to process the input and get the final embedding layer.</li>
</ol>
<h2 id="positional-encoding">Positional encoding<a class="anchor" aria-hidden="true" hidden href="#positional-encoding">#</a></h2>
<p>We need to keep positional information about the contents.</p>
<blockquote>
<p>two randomly chosen uncorrelated vectors tend to be nearly orthogonal in spaces of
high dimensionality, indicating that the network is able to process the token identity
information and the position information relatively separately.</p>
</blockquote>
<h4 id="binary-positional-encoding">Binary positional encoding<a class="anchor" aria-hidden="true" hidden href="#binary-positional-encoding">#</a></h4>
<p>This is just a simple idea to encode the information about the position of the tokens.

<h4 id="th-high-dimensional-unit-vectors-are-almost-always-orthogonal">Th: High dimensional unit vectors are almost always orthogonal<a class="anchor" aria-hidden="true" hidden href="#th-high-dimensional-unit-vectors-are-almost-always-orthogonal">#</a></h4>
<p>This theorem states that given <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.8889em;vertical-align:-0.1944em;"></span><span <h4 id="building-the-embeddings">Building the embeddings<a class="anchor" aria-hidden="true" hidden href="#building-the-embeddings">#</a></h4>
<p>Usually, the embeddings for each vector are <a href="/notes/backpropagation/">backpropagated</a> during training.
In Pytorch, you would use <code>nn.Embedding</code>. In this manner, each token is assigned a vector of size <code>embedding</code> chosen by the designer of the architecture.
This value is then initialized, and every index of the vector is a learnable parameter.</p>
<h2 id="attention">Attention<a class="anchor" aria-hidden="true" hidden href="#attention">#</a></h2>
<p>First introduced in <a href="http://arxiv.org/abs/1409.0473">(Bahdanau et al. 2014)</a> in the context of translation.</p>
<blockquote>
<p>Whereas standard networks multiply activations by fixed weights, here the activations are multiplied by the data-dependent attention coefficients.</p>
</blockquote>
<blockquote>
<p><strong>Soft attention</strong> in which we use continuous variables to measure the degree of match between queries and keys and we then use these variables to weight the influence of the value vectors on the outputs.</p>
</blockquote>
<p>Its main objective is to <em>identify useful previous information</em>.</p>
<h4 id="the-desiderata">The desiderata<a class="anchor" aria-hidden="true" hidden href="#the-desiderata">#</a></h4>
<p>We have said that transformers transform a set of data points into another with the same dimensionality. Attention is one important part of the architecture.
To have richer representation space, we need each entry of the new vector to be a linear combination of the input.
Plus, we need two properties for the attention coefficients:</p>
<ul>
<li>They should be positive or null (you pay some attention, or none, it doesn't make sense to have negative attention!)</li>
<li>It should be limited (e.g. their sum should be 1), we choose one so that we don't make the variance of the input explode.</li>
</ul>
<h4 id="the-intuition">The intuition<a class="anchor" aria-hidden="true" hidden href="#the-intuition">#</a></h4>
<p>Attention is an architecture used in <a href="/notes/transformers/">Transformers</a> to encode a <strong>soft</strong> version of dictionaries.
In the context of text classification, the main intuition is giving a certain <em>weight</em> of some tokens, probably contextually more important, and less than others.
If we have a query, which is something we would like to know about the text, then we try to match it with a key and the relative value. The softness of attention prevents us to say: "if the key doesn't match just return error", instead, it returns a linear combination of possible values, accordingly weighted by the rescaled keys.</p>
<p>Intuitively in the text context, attention models how much the value of one token influences another, <em>directionally</em>.</p>
<h4 id="on-the-asymmetricity">On the Asymmetricity<a class="anchor" aria-hidden="true" hidden href="#on-the-asymmetricity">#</a></h4>
<blockquote>
<p>For example, we might expect that ‘chisel’ should be strongly associ-
ated with ‘tool’ since every chisel is a tool, whereas ‘tool’ should only be weakly
associated with ‘chisel’ because there are many other kinds of tools besides chis-
els.</p>
</blockquote>
<p>Asymmetric matrices have a higher relation representation capacity as we see from the above example.
This motivates a different matrix for queries keys and values.</p>
<h4 id="self-attention">Self attention<a class="anchor" aria-hidden="true" hidden href="#self-attention">#</a></h4>
<blockquote>
<p>Compared to a conventional neural network, the signal paths have multiplicative relations between activation values. Whereas standard networks multiply activations by fixed weights, here the activations are multiplied by the data-dependent attention coefficients.</p>
</blockquote>

<h4 id="clip-model">CLIP Model<a class="anchor" aria-hidden="true" hidden href="#clip-model">#</a></h4>
<p>We want to be able to produce image from caption and caption from image. So this is just some joint pretraining of captioned images, and it is able to do it well. Uses cosine similarity to do it.</p>
<p>One nice thing is using caption to classify the image, also better compared to some specific image classification methods.</p>
<h4 id="dinov2">DINOv2<a class="anchor" aria-hidden="true" hidden href="#dinov2">#</a></h4>
<p>CLIP models are limited by its training dataset, that needs pairs of images and texts.
DINOv2 provides some features: it is <strong>self-distillation with no features</strong>. But here we don't have a teacher network -> idea to build it from past iterations of the student.
Using DINO works better with other self-supervised methods, and even better to some supervised methods, probably it creates better features.</p>
<h4 id="4m-massively-multimodal-masked-modeling">4M: Massively multimodal Masked Modeling<a class="anchor" aria-hidden="true" hidden href="#4m-massively-multimodal-masked-modeling">#</a></h4>
<p>They basically project everything into a common embedding space:</p>
<ul>
<li>For text they use wordpiece</li>
<li>For images they use VQ-VAE (see <a href="/notes/autoencoders/">Autoencoders</a>)</li>
<li>For bounding boxes they use sequential prediction, similar to pix2seq.
Then they train a single masked model to predict all the modalities.</li>
</ul>
