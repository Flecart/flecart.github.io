One of the main trends is <strong>disaggregation</strong>: we want to be able to scale different resources independently.</p>
<h4 id="introduction-to-cxl-compute-express-link">Introduction to CXL (Compute Express Link)<a class="anchor" aria-hidden="true" hidden href="#introduction-to-cxl-compute-express-link">#</a></h4>
<p>This is a new part of the memory hierarchy.</p>
<p><img src="/images/notes/Compute Express Link-20250518115846502.webp" style="width: 100%" class="center" alt="Compute Express Link-20250518115846502"></p>
<p>NVM is a kind of non volatile memory that is used as a storage device that is close to the device (others are network attached or slower than network part anyways. It is persistent and has low latency. It is used in the memory hierarchy to extend the memory capacity.</p>
<p>The main advantages are:</p>
<ul>
<li>Independent scaling of memory and compute</li>
<li>Improved resource utilization</li>
<li>Enhanced fault tolerance</li>
</ul>
<p>Disadvantages are:</p>
<ul>
<li>Complexity in coherence and consistency, and synchronization.</li>
</ul>
<p>In cloud systems usually is important to individuate the <strong>Hyper scalers</strong> which is the app, or group of apps, that are taking the most resources in the cloud.</p>
<h4 id="cxl-protocols">CXL protocols<a class="anchor" aria-hidden="true" hidden href="#cxl-protocols">#</a></h4>
<p>PCIe connects devices to the CPU. It is a <strong>point-to-point</strong> connection usually used for FPGAs, GPU and network parts to the CPU.
CXL extends the PCIe for heterogenous computing and server disaggregation usages</p>
<ul>
<li>CXL.io: based on PCIe, it does the same thing basically (device discovery, DMA, I/O virtualization and DMA).</li>
<li>CXL.cache: this is a cache coherent protocol, it allows the CPU to access memory from other devices.</li>
<li>CXL.mem: this is a memory coherent protocol, it allows the CPU to access memory from other devices.</li>
</ul>
<h4 id="cxl-devices">CXL Devices<a class="anchor" aria-hidden="true" hidden href="#cxl-devices">#</a></h4>
<p>There are three types of devices</p>
<ul>
<li>CXL Type 1: Accelerator devices, they are used for compute acceleration. They can access memory from the CPU and other devices. (io and cache, sits at the place of standard coherence protocols)</li>
<li>CXL Type 2: Accelerator devices that need all the protocols.</li>
<li>CXL Type 3: Memory devices, they are memory devices that can be accessed by the CPU and other devices. They are used for memory expansion. (io and mem)</li>
</ul>
<h3 id="on-memory-coherence">On Memory Coherence<a class="anchor" aria-hidden="true" hidden href="#on-memory-coherence">#</a></h3>
<p>Memory coherence is a property of a system that ensures that <em>all copies of a memory location are consistent</em>. This is important for parallel computing, where multiple processors can access the same memory location.</p>
<p>For example, if you have two CPUs that modify a shared cache line, you need to ensure that the cache line is consistent across all CPUs. This is done by using a <em>cache coherence protocol</em>, which ensures that all copies of the cache line are consistent.</p>
<h4 id="mesi-protocol">MESI protocol<a class="anchor" aria-hidden="true" hidden href="#mesi-protocol">#</a></h4>
<p>One of the most famous protocols is <strong>MESI</strong>:
If you want to read x, you go from invalid to shared state
If you want to write x, you go to exclusive state and after write it is modified state. All can be represented by a state diagram, see <a href="/notes/grammatiche-regolari/">Grammatiche Regolari</a>.
<img src="/images/notes/Compute Express Link-20250518121014313.webp" style="width: 100%" class="center" alt="Compute Express Link-20250518121014313"></p>
<h4 id="snoop-filters">Snoop Filters<a class="anchor" aria-hidden="true" hidden href="#snoop-filters">#</a></h4>
<p>It is implemented through <strong>snooping</strong>, that act as gatekeeper for bus snooping. It tracks which caches hold copies of specific cache blocks and only
broadcasts snoop requests when they are truly needed.</p>
<h4 id="directory-controller">Directory Controller<a class="anchor" aria-hidden="true" hidden href="#directory-controller">#</a></h4>
<p>It is a centralized structure that tracks the state of each cache line in the system. It is used to reduce the amount of snooping traffic on the bus. It is used in large systems with many caches.</p>
<h4 id="coherence-in-cxl">Coherence in CXL<a class="anchor" aria-hidden="true" hidden href="#coherence-in-cxl">#</a></h4>
<p>We want to have a coherence across all the devices (non only cache, but also accelerators!) attached to our system</p>
<p>The CXL.mem is able to</p>
<ul>
<li>Version 1.0: memory expansion (machines are able to split the memory into zones that are responsibility of some specific devices).</li>
<li>Version 2.0: memory pooling</li>
<li>Version 3.2: memory sharing, but not implemented yet (it is far more complex, and not clear if it will be helpful).</li>
</ul>
<p>This is some sort of <strong>asymmetric protocol</strong>, I don't know what it means.</p>
