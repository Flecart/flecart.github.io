1d:T183a5,<p>Log Linear Models can be considered the most basic model used in natural languages. The main idea is to try to model the correlations of our data, or how the posterior <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal">p</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0359em;">y</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">∣</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal">x</span><span class="mclose">)</span></span></span></span> varies, where <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.4306em;"></span><span class="mord mathnormal">x</span></span></span></span> is our single data point features and <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.625em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0359em;">y</span></span></span></span> are the labels of interest. This is a <em>form of generalization</em> because contextualized events (x, y) with similar descriptions tend to have similar probabilities.</p>
<p>These kinds of models are so common that it has been discovered in many fields (and thus assuming different names): some of the most famous are Gibbs distributions, undirected graphical models, Markov Random Fields or Conditional Random Fields, exponential models, and (regularized) maximum entropy models. Special cases include logistic regression and Boltzmann machines.</p>

<h3 id="feature-pipeline">Feature pipeline<a class="anchor" aria-hidden="true" hidden href="#feature-pipeline">#</a></h3>
<h4 id="the-preprocessing-steps">The preprocessing steps<a class="anchor" aria-hidden="true" hidden href="#the-preprocessing-steps">#</a></h4>
<p>Historically, the features were <em>hand-engineered</em>, meaning people would try to extract functions after a step of preprocessing the text. the preprocessing step consisted in steps like</p>
<ul>
<li>Tokenization</li>
<li>Lower casing</li>
<li>stemming (getting the root of the words)</li>
<li>Stop word removal</li>
<li>Reducing vocabulary.
Here is an example:
<img src="/images/notes/Log Linear Models-20240904212409237.webp" style="width: 100%" class="center" alt="Log Linear Models-20240904212409237">
And then the input would be easy enough to be dealt with the log-linear method and the features.</li>
</ul>
<h4 id="feature-engineering">Feature engineering<a class="anchor" aria-hidden="true" hidden href="#feature-engineering">#</a></h4>
<p>After we have preprocessed the texts, how can we define the features? There are some classical methods:
N-grams</p>
<ul>
<li>(BOS, BOS, UNK), (BOS, UNK, ‘s), ..., (complete, broke, !
One-hot encoding</li>
<li>(0, 0, 1, 0, ..) for a single word.
Bag-of-words (see spam classification)</li>
<li>(0, 1, 0, 2, ...), we don't care about the order of the words, just if they are present or not in the text.
Word embeddings</li>
<li>Convert words into continuous representations</li>
<li>Can be contextual (e.g., ELMo, BERT) or not (e.g., Glove, fastText)</li>
<li>More on this in future lectures!
Bag-of-embeddings</li>
<li>Average embeddings for all words in a sequence
Domain-specific features (for example if we know we want to agree the gender of an adjective, this feature could be built-in!)</li>
</ul>
<p>After we have defined this, we use <a href="/notes/parametric-modeling/#the-maximum-likelihood-estimator">MLE</a> to minimize the loss. In the case of log linear models, it is a convex function, so we have that the local minima is also the global minima, and it's a easy problem to solve.</p>
<p>With the new approach, features are designed automatically with neural networks.</p>

<h4 id="relation-with-linear-models">Relation with linear models<a class="anchor" aria-hidden="true" hidden href="#relation-with-linear-models">#</a></h4>
<p>One can prove that log linear models can learn any linear model. The simple reason is that in a binary-class log-linear model the decision boundary is given by <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal" style="margin-right:0.0278em;">θ</span><span class="mord mathnormal" style="margin-right:0.0359em;">g</span><span class="mopen">(</span><span class="mord mathnormal">x</span><span class="mclose">)</span></span></span></span>, and falls back to a Sigmoid function, this is the same as the linear model bound!</p>
