29:T2658,<h4 id="what-is-reflection">What is Reflection?<a class="anchor" aria-hidden="true" hidden href="#what-is-reflection">#</a></h4>
<p>Reflection enables a program to observe and modify its own structure and behavior. t bridges the gap between the <strong>base level</strong> (the application logic) and the <strong>meta level</strong> (objects representing the program itself).</p>
<!--SR:!2025-12-19,9,250-->
<h4 id="core-concepts">Core Concepts<a class="anchor" aria-hidden="true" hidden href="#core-concepts">#</a></h4>
<ul>
<li><strong>Introspection</strong>: The program can observe its own type information at run time.</li>
<li><strong>Intercession</strong> (Dynamic Manipulation): The program can modify its own structure and behavior (e.g., adding methods, changing fields).</li>
<li><strong>Reflective Code Generation</strong>: Creating executable code from data structures at run time.</li>
</ul>
<!--SR:!2025-12-12,2,230-->
<h3 id="81-introspection"><strong>8.1 Introspection</strong><a class="anchor" aria-hidden="true" hidden href="#81-introspection">#</a></h3>
<p>Introspection is the ability to examine the runtime type of an object. The entry point for this in Java is the <code>Class</code> object.</p>
<h4 id="obtaining-class-objects">Obtaining Class Objects<a class="anchor" aria-hidden="true" hidden href="#obtaining-class-objects">#</a></h4>
<p>There are three main ways to get a <code>Class</code> object in Java:</p>
<ol>
<li><strong>Object instance</strong>: <code>myObject.getClass()</code></li>
<li><strong>Class literal</strong>: <code>String.class</code></li>
<li><strong>String name</strong>: <code>Class.forName("java.lang.String")</code> (Can throw runtime errors)</li>
</ol>
<!--SR:!2025-12-12,2,230-->
<h4 id="inspecting-members"><strong>Inspecting Members</strong><a class="anchor" aria-hidden="true" hidden href="#inspecting-members">#</a></h4>
<p>Once you have the <code>Class</code> object, you can retrieve its members.</p>
<ul>
<li><code>getMethods()</code> / <code>getFields()</code>: Returns <strong>public</strong> members (including inherited ones).</li>
<li><code>getDeclaredMethods()</code> / <code>getDeclaredFields()</code>: Returns <strong>all</strong> members declared in the class (including private ones, but <em>excluding</em> inherited ones).</li>
</ul>
<!--SR:!2025-12-12,2,230-->
<blockquote>
<p>[cite_start]<strong>Safety &#x26; Access</strong>: To access private members, you must suppress Java's access checking using <code>setAccessible(true)</code>[cite: 147]. This breaks information hiding and encapsulation.</p>
</blockquote>
<h4 id="example-universal-tostring"><strong>Example: Universal <code>toString</code></strong><a class="anchor" aria-hidden="true" hidden href="#example-universal-tostring">#</a></h4>
<p>A generic <code>toString</code> method can be written using reflection to iterate over all fields of any object.</p>
<pre><code class="language-java">public static String getString(Object o) {
    if (o == null) return "null";
    Class c = o.getClass();
    // We use getDeclaredFields to see private state
    Field[] fields = c.getDeclaredFields(); 
    for (Field f : fields) {
        if (!Modifier.isStatic(f.getModifiers())) {
             f.setAccessible(true); // Bypass private check
             // ... read value ...
        }
    }
}
</code></pre>
<p>Safety checks (like <code>IllegalAccessException</code>) are moved from compile time to run time1111.</p>
<h4 id="dynamic-invocation-unit-testing"><strong>Dynamic Invocation (Unit Testing)</strong><a class="anchor" aria-hidden="true" hidden href="#dynamic-invocation-unit-testing">#</a></h4>
<p>Reflection allows invoking methods when their names are not known at compile time.</p>
<ul>
<li><strong>Mechanism</strong>: <code>Method.invoke(Object receiver, Object... args)</code>2.</li>
<li><strong>Use Case (JUnit)</strong>: Test drivers can find all methods starting with "test" and invoke them dynamically. This decouples the test runner from the specific test classes3.</li>
</ul>
<h4 id="reflective-visitor-pattern">Reflective Visitor Pattern<a class="anchor" aria-hidden="true" hidden href="#reflective-visitor-pattern">#</a></h4>
<p>Standard Visitor relies on accept methods in every element class. The Reflective Visitor uses introspection to dispatch calls, removing the need for accept4444.</p>
<ul>
<li><strong>Mechanism</strong>: The visitor constructs the method name dynamically based on the element's class.
<pre><code class="language-java">// "magic" string construction
String name = "visit" + e.getClass().getName();
Method m = this.getClass().getMethod(name, e.getClass());
m.invoke(this, e);
</code></pre>
</li>
</ul>
<!--SR:!2025-12-21,11,270-->
<ul>
<li><strong>Pros</strong>: Simpler code in element classes, flexible lookup.</li>
<li><strong>Cons</strong>: Not statically safe (runtime crash if method missing), slower performance.</li>
</ul>
<h4 id="generics-and-erasure">Generics and Erasure<a class="anchor" aria-hidden="true" hidden href="#generics-and-erasure">#</a></h4>
<p>Due to Java's erasure semantics, generic type parameters are <strong>not</strong> available via reflection on instances.</p>
<ul>
<li>Example: You cannot distinguish <code>List&#x3C;String></code> from <code>List&#x3C;Integer></code> at runtime.</li>
<li><code>getMethod("add", String.class)</code> will fail on a <code>List&#x3C;String></code> because the compiled method signature expects <code>Object</code>, not <code>String</code>8888.</li>
</ul>
<!--SR:!2025-12-19,9,250-->
<h3 id="82-reflective-code-generation"><strong>8.2 Reflective Code Generation</strong><a class="anchor" aria-hidden="true" hidden href="#82-reflective-code-generation">#</a></h3>
<p>This involves creating executable code from data structures at runtime9.</p>
<h4 id="why-reflection">Why Reflection?<a class="anchor" aria-hidden="true" hidden href="#why-reflection">#</a></h4>
<p>If code is represented as data, programs can create code from data. This allows adapting the program to user input or the environment (e.g., generating a specific SQL query based on a user filter)10.</p>
<!--SR:!2025-12-19,9,250-->
<h4 id="c-expression-trees">C# Expression Trees<a class="anchor" aria-hidden="true" hidden href="#c-expression-trees">#</a></h4>
<p>In C#, code (specifically lambda expressions) can be treated as data structures called <strong>Expression Trees</strong>11.</p>
<ul>
<li><strong>Structure</strong>: Trees composed of <code>BinaryExpression</code> (operators), <code>ParameterExpression</code> (variables), etc.12.</li>
<li><strong>Compilation</strong>: A <code>.Compile()</code> method turns the tree data structure into executable code (a delegate) at runtime.</li>
</ul>
<pre><code class="language-c#">// Builds the tree for: x &#x3C; c
Expression.Lambda&#x3C;Func&#x3C;int, bool>>(
    Expression.LessThan(lhs, ce), // The binary expression node
    new ParameterExpression[] { lhs }
).Compile(); // Transforms tree -> executable function
</code></pre>
<h3 id="83-dynamic-code-manipulation"><strong>8.3 Dynamic Code Manipulation</strong><a class="anchor" aria-hidden="true" hidden href="#83-dynamic-code-manipulation">#</a></h3>
<p>This refers to modifying the structure of a program (adding/removing methods) while it is running14.</p>
<h4 id="python-example"><strong>Python Example</strong><a class="anchor" aria-hidden="true" hidden href="#python-example">#</a></h4>
<p>Dynamic languages like Python allow extensive manipulation ("Intercession").</p>
<ul>
<li><strong>Monkey Patching</strong>: You can replace methods on a class at runtime15.</li>
<li><em>Example</em>: A generic wrapper can be applied to all methods of a class to make them case-insensitive16.</li>
<li><strong>Mechanism</strong>: Methods can be added, exchanged, or deleted using <code>setattr</code> and <code>del</code>17171717.</li>
</ul>
<pre><code class="language-python"># Injecting a new method into a class at runtime
def new_method(self): return "patched"
setattr(MyClass, "method_name", new_method)
</code></pre>
<h3 id="reflection-and-type-checking"><strong>Reflection and Type Checking</strong><a class="anchor" aria-hidden="true" hidden href="#reflection-and-type-checking">#</a></h3>
