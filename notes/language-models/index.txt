28:T11b93,<p>In order to understand language models we need to understand <strong>structured prediction</strong>. If you are familiar with <a href="/notes/sentiment-analysis/">Sentiment Analysis</a>, where given an input text we need to classify it in a binary manner, in this case the output space usually scales in an <em>exponential</em> manner. The output has some structure, for example it could be a tree, it could be a set of words etc... This usually needs an intersection between statistics and computer science.</p>

<h3 id="definition-of-a-language-model">Definition of a language model<a class="anchor" aria-hidden="true" hidden href="#definition-of-a-language-model">#</a></h3>
<p>A language model is defined as a distribution over a <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6887em;"></span><span class="mord"><span class="mord">Σ</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height:0.6887em;"><span style="top:-3.063em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mtight">∗</span></span></span></span></span></span></span></span></span></span></span></span> (See <a href="/notes/descrizione-linguaggio/">Descrizione linguaggio</a> for the Kleene star and the alphabet).
So it <em>assigns probabilities</em> to <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.7335em;vertical-align:-0.1944em;"></span><span class="mord mathnormal" style="margin-right:0.0359em;">y</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">∈</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6887em;"></span><span class="mord"><span class="mord">Σ</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height:0.6887em;"><span style="top:-3.063em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mtight">∗</span></span></span></span></span></span></span></span></span></span></span></span>. Remember: every element in <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6887em;"></span><span class="mord"><span class="mord">Σ</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height:0.6887em;"><span style="top:-3.063em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mtight">∗</span></span></span></span></span></span></span></span></span></span></span></span> is <strong>finite</strong>. <p>But we need to be careful about some details. As <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6887em;"></span><span class="mord"><span class="mord">Σ</span><span class="msupsub"><span class="vlist-t"><span class="vlist-r"><span class="vlist" style="height:0.6887em;"><span style="top:-3.063em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mtight">∗</span></span></span></span></span></span></span></span></span></span></span></span> is an infinite set it's not trivial to define a probability distribution over it. This motivates us to have two other definitions</p>

<h4 id="tightness-of-language-models">Tightness of language models<a class="anchor" aria-hidden="true" hidden href="#tightness-of-language-models">#</a></h4>
<p>With a <strong>tight</strong> locally normalized language model we have the sum of probability of every path sums to 1.
With non-tight it is possible that we lose some probability to infinite structures.
It is possible to impose tightness: just set <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord mathnormal">p</span><span class="mopen">(</span><span class="mord mathnormal" style="margin-right:0.0576em;">E</span><span class="mord mathnormal" style="margin-right:0.0278em;">O</span><span class="mord mathnormal" style="margin-right:0.0576em;">S</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">∣</span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:1em;vertical-align:-0.25em;"></span><span class="mord text"><span class="mord"> parent</span></span><span class="mclose">)</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">></span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.5782em;vertical-align:-0.0391em;"></span><span class="mord mathnormal">ε</span><span class="mspace" style="margin-right:0.2778em;"></span><span class="mrel">></span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">0</span></span></span></span> for some <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.4306em;"></span><span class="mord mathnormal">ε</span></span></span></span>.</p>
<p>It's very easy to find applications for conditional language modelling, some examples are:</p>
<ul>
<li>In machine translation, is source-language text, and is the target-language text</li>
<li>In speech recognition, is an audio signal, and is target-language text</li>
<li>In summarization, is a longer text, and is a shorter text</li>
<li>In dialogue systems, is a user’s input and is the agent’s response</li>
</ul>
<p>In modern systems the following three models are the most famous ones for defining Language Models: n-gram models, recurrent neural networks and <a href="/notes/transformers/">Transformers</a>.</p>

<p>It's very easy to make n-gram language models tight, we just need the probability of <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6833em;"></span><span class="mord mathnormal" style="margin-right:0.0576em;">E</span><span class="mord mathnormal" style="margin-right:0.0278em;">O</span><span class="mord mathnormal" style="margin-right:0.0576em;">S</span></span></span></span> for every history to be <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.5782em;vertical-align:-0.0391em;"></span><span class="mrel">></span><span class="mspace" style="margin-right:0.2778em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">0</span></span></span></span>.</p>

<h4 id="some-observations">Some observations<a class="anchor" aria-hidden="true" hidden href="#some-observations">#</a></h4>
<p>With the n-gram assumption, if we consider 1-grams they behave very much like bag of words, they just assign probability to single words, regardless of the order where they appear.
Other problems with n-grams are with sparsity of the data: if a n-gram never appears, its probability is 0, which is too hard as a requirement for language modelling.</p>
<h3 id="a-naive-implementation">A naive implementation<a class="anchor" aria-hidden="true" hidden href="#a-naive-implementation">#</a></h3>
<p>A simple solution is just define a <em>different probability distribution for every possible context</em> of size <span class="katex"><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height:0.6667em;vertical-align:-0.0833em;"></span><span class="mord mathnormal">n</span><span class="mspace" style="margin-right:0.2222em;"></span><span class="mbin">−</span><span class="mspace" style="margin-right:0.2222em;"></span></span><span class="base"><span class="strut" style="height:0.6444em;"></span><span class="mord">1</span></span></span></span>. <h3 id="the-neural-n-gram-language">The Neural n-gram language<a class="anchor" aria-hidden="true" hidden href="#the-neural-n-gram-language">#</a></h3>

class="vlist" style="height:0.2806em;"><span style="top:-2.55em;margin-left:0em;margin-right:0.05em;"><span class="pstrut" style="height:2.7em;"></span><span class="sizing reset-size6 size3 mtight"><span class="mord mtight"><span class="mord mathnormal mtight">t</span></span></span></span></span><span class="vlist-s">​</span></span><span class="vlist-r"><span class="vlist" <p>Similar idea to <a href="/notes/log-linear-models/">Log Linear Models</a> discussed in previous lessons, applied again in different manner.
This gives us just the parameters to learn the embedding, which is free of choice for us.</p>
<h4 id="the-architecture">The architecture<a class="anchor" aria-hidden="true" hidden href="#the-architecture">#</a></h4>
<p>This model was proposed in <a href="https://papers.nips.cc/paper_files/paper/2000/hash/728f206c2a01bf572b5940d7d9a8fa4c-Abstract.html">(Bengio et al. 2003)</a> and gave state of the art performance. But is was slow. So this model is <strong>not scalable</strong>. And from the practical point of view it is almost useless. Nonetheless the ideas introduced here are of paramount historical and scientific importance.</p>
<p><img src="/images/notes/Language Models-20240908123344732.webp" style="width: 100%" class="center" alt="Language Models-20240908123344732"></p>
<p>This is the main idea</p>
<blockquote>
<p>If we use an MLP with shared parameters, we can reduce the number of parameters and, hopefully, increase the ability of the model to generalize</p>
</blockquote>
<p>We are using a single MLP to estimate many many conditional distributions!</p>
<h4 id="summary-of-the-key-points">Summary of the key points<a class="anchor" aria-hidden="true" hidden href="#summary-of-the-key-points">#</a></h4>
<p>As described in section 1.1 of <a href="https://papers.nips.cc/paper_files/paper/2000/hash/728f206c2a01bf572b5940d7d9a8fa4c-Abstract.html">(Bengio et al. 2003)</a>, the whole idea behind this method can be summarized in three clean points:</p>
