Axiomatic Approach to Entropy#

We can define the entropy as the only function that satisfies the following properties (the function is commonly called surprise):

  1. H(X)≥0 for every discrete r.v. which is related to the code length, which we don't want to be negative
  2. H(X,Y)=H(X)+H(Y) when X,Y are independent random variables.
  3. H(X) is maximal  when X∼Unif(0,1) this concept is related to hard to guess the results.

Then one can prove that the only function that satisfies these properties is the entropy. Which is:

H(X)=−x∈X∑​p(x)logp(x)

Properties of the entropy#

One observation is that labels don't matter, we just need the probability vector and don't care about what it represents.

The entropy is always positive. It's easy to prove because all probabilities are 0<p≤1 so every term in the sum is positive because log in that interval is also positive.

Axiomatic approach#

One could derive the entropy from an axiomatic point of view, with the idea of Surprise We just need three requirements: Given a probability space Ω,A,P , then the surprise of an even E⊆A is equal to a function S([P(E)]) which satisfies the following properties:

  1. S(1)=0
  2. S is continuous
  3. S is monotonic decreasing, meaning that if p>q then S(p)<S(q)
  4. S is additive, meaning that S(pq)=S(p)+S(q) when p,q are independent.

Then we can prove that the only function satisfying these properties has the form:

S(p)=−log(p)

The entropy is just the expected value of the surprise.

Monotonicity of Entropy#

This theorem states that

H(X)≥H(X∣Y)

It is also called the principle Information never hurts meaning you are never more uncertain about a random variable when you have more information about it.

It's easy to see why its true!

H(X)−H(X∣Y)=∑P(x)logP(x)−∑P(x,y)logP(x∣y)=∑P(x,y)logP(x)P(x,y)​Jensen≥​​0

This is also the reason why mutual information is always positive.

Relative Entropy or Kullback-Leibler#

D(P∣∣Q)=x∈X∑​P(x)logQ(x)P(x)​

We need to define some corner cases:

  • If P(x)=0 and Q is anything then its 0
  • If ∃ξ∈X such that P(ξ)>0 and Q(ξ)=0 => D(P∣∣Q)=+∞.
  • If P≪Q then D(P∣∣Q)<∞ (I did not understand why)

This has some relations with the entropy, we can use some log properties and have the following result:

D(P∣∣Q)=−H(P)−∑P(x)logQ(x)

The second addendum can be called cross-entropy.

In modo praticamente equivalente possiamo definire una versione condizionata. e si può applicare anche in questo caso una chain rule

DL(P(x,y)∣∣Q(x,y)=DL(P(x)∣∣Q(x))+DL(P(x∣y)∣∣Q(x∣y))

Relative entropy is not a distance not a metric, so its incorrect to say it is a distance, but for practical purposes it seems to work well: if the Relative entropy is small also the probability vectors are small.

Relationship with χ2-divergence.#

Chi-squared divergence — χ²(P‖Q) = ∫ (p(x)−q(x))²/q(x) dx = E_q[(p/q)²] − 1 — is an f-divergence with f(t) = (t−1)². It sits in an interesting relationship with KL: it upper bounds KL (KL(P‖Q) ≤ log(1+χ²(P‖Q)) ≤ χ²(P‖Q)), and locally both reduce to the Fisher information metric to leading order.

Pros

Stronger divergence, better bounds. Because χ² dominates KL, minimizing it gives you a KL bound for free. This shows up usefully in PAC-Bayes (Catoni-style bounds), DRO (Duchi–Namkoong's chi-squared uncertainty balls have clean duals and finite-sample guarantees), and in CHIVI (Dieng et al. 2017), where you get an upper bound on log evidence — complementing the ELBO's lower bound so you can sandwich it.

Direct variance interpretation. χ²(p‖q) is the variance of the importance ratio p/q (up to the −1). So if your downstream use of q is importance sampling — off-policy RL, Monte Carlo estimation, self-normalized estimators — chi-squared minimization is literally variance minimization on the estimator. KL doesn't give you that.

More mode-covering (in the forward direction). Reverse KL is famously mode-seeking; forward KL is mean-seeking but logarithmically forgiving of mass-coverage failures. Chi-squared penalizes those failures quadratically, so it pushes harder to cover tails of p.

Doesn't require log-densities. Useful when you only have unnormalized densities or numerically fragile log-probs, and in some f-GAN-style objectives. LSGAN is essentially a chi-squared variant.

Robustness properties. Chi-squared-regularized policy optimization has been shown to be more robust to model misspecification than KL-regularized versions in some RL settings — related to its DRO interpretation.

Cons

Catastrophically high-variance estimators. This is the big one. The integrand involves p²/q, so whenever q assigns low probability where p doesn't, the Monte Carlo estimator explodes. KL with its log ratio is far more numerically forgiving. In practice this is why chi-squared methods often fail to scale.

Exponential blow-up with dimension. KL is additive over independent factors: KL(p₁p₂‖q₁q₂) = KL(p₁‖q₁) + KL(p₂‖q₂). Chi-squared is multiplicative: 1+χ² = (1+χ²₁)(1+χ²₂). So for d independent dims with per-dim χ² = c, the joint χ² is (1+c)^d − 1. For high-dimensional models (sequences, images, multi-agent state spaces) this is a real problem.

No MLE interpretation. Forward KL minimization is maximum likelihood, which gives you all of statistical learning theory for free. Chi-squared minimization is a different estimator — consistent, but without the same asymptotic and finite-sample machinery.

Loses the exponential-family / Bregman structure. KL between exponential families is a Bregman divergence with closed forms; gradients and natural gradients have clean expressions. Chi-squared lacks this, so you give up a lot of conjugacy and tractability.

Less mature tooling and theory. Most autodiff frameworks, RL algorithms, and VI methods are built around KL/cross-entropy. Convergence theory, hyperparameter heuristics, and best practices are far thinner for chi-squared, so you're often doing more methodological work yourself.

Sensitive to support mismatch. If p has any mass where q is zero, χ² is infinite — same as KL — but the quadratic penalty makes near-zero q values numerically catastrophic well before you hit the singularity.

When it's actually worth it

Roughly: when you specifically care about variance of importance weights (off-policy estimation, sequential Monte Carlo), when you need an upper bound on log evidence (CHIVI), when you want DRO guarantees, or in low-dimensional problems where the variance and dimensionality issues don't bite. For typical large-scale training — RLHF, VI on neural nets, language modeling — KL's numerical behavior and additivity make it the default for good reasons, and chi-squared mostly shows up as a regularizer or auxiliary term rather than the primary objective.

Mutual information#

Questa nozione definisce quanta informazione hanno in comune due variabili aleatorie

Definizione#

I(X;Y)=x∑​y∑​p(x,y)log(p(x)p(y)p(x,y)​)=H(X)−H(X∣Y)

Si può fare dopo un po' di calcoli che qui ho omesso, ma non dovrebbe essere difficile farlo.888

Si può intendere la mutual information anche come KL fra le distribuzioni p(x,y) e p(x)p(y) si può notare che queste due sono uguali quando le due sono indipendenti, che è coerente con la nostra nozione che abbiamo dell'indipendenza.

Proprietà#

Another property:

I(X;Y∣Z)=I(X;Y,Z)−I(X;Z)

Redundancy and Synergy#

If we consider the interaction information

I(X;Y;Z)=I(X;Y)−I(X;Y∣Z)

Then synergy between Y and Z is present when the interaction information is negative, and redundant when its positive. This is because, intuitively, if it is negative it means Z can provide more information with Y about Z, else, they both have some information about X which is probably not enough.

Example: Independent Variable in MI#

We see that if Y∣X⊥Z then I(X;Z)=I(X,Y;Z) this means then that the relation between Y and Z is neither redundant or synergic

I(X,Y;Z)​=E[logP(X,Y)P(Z)P(X,Y,Z)​]=E[logP(X)P(Z)P(X,Z)​]+E[logP(Y∣X)P(Y∣X,Z)​]=I(X;Z)+E[logP(Y∣X)P(Y∣X,Z)​]=I(X;Z)​​

Where in the last step we used the fact that Y⊥Z∣X⟹P(Y∣X,Z)=P(Y∣X).

This means that if the independence condition is satified, then we can add random variables as we like without changing the mutual information.

Possiamo rappresentare il sampling da una certa famiglia di distribuzioni fθ​(x) , rappresentato da X, e una sua statistica a caso (media varianza etc, che credo basti una funzione sul valore) come T, allora possiamo rappresentarlo come una Markov Chains#Catena di 3 variabili θ→X→T(X) E vale il teorema di information processing

Si può chiamare una statistica per θ sufficiente se X contiene tutta l'informazione di θ. Non so bene cosa significhi. La cosa importante è che la statistica sufficiente preserva la mutua informazione ossia si ha una uguaglianza in quella relazione di sopra. Vedere 2.9 di (Cover & Thomas 2012) per esempi .

Questo è un teorema fondamentale per moltissime cose, e da un certo punto di vista è una cosa banale per le cose convesse/concave. Allora, sia data una funzione in [a,b] tale che sia convessa (concava) in questo intervallo, allora vale che

f(i∑​λi​xi​)≤i∑​λi​f(xi​)

Language Models#

Entropy Rate is not Constant#

Paper: Braverman et al. (2020) - "Calibration, Entropy Rates, and Memory in Language Models" This seminal ICML 2020 paper provides definitive empirical evidence against entropy rate constancy.

Key Finding: State-of-the-art language models exhibit dramatic entropy rate amplification during generation.

Empirical Evidence:

ModelTest PerplexityGenerated Text Perplexity
AWD-LSTM (PTB)58.393.1
CNN-LSTM (GBW)29.849.4
Transformer (GBW)28.134.7
GPT-2 (WebText)23.761.2

Interpretation:

  • Test perplexity measures the model's one-step prediction accuracy
  • Generated text perplexity measures exp(entropy rate) of the model's own generations
  • The gap shows that entropy rate increases dramatically over time during generation
  • For GPT-2: entropy rate grows from ~23.7 to ~61.2 (2.6x increase!)

2.2 Visual Evidence#

The paper shows perplexity curves (exp of conditional entropy) over generation time:

  • At t=1: Model's upper bound for language perplexity (test set performance)
  • As t→∞: Exponential of the entropy rate of the model's generations
  • Result: Curves drift upward monotonically, not flat

For a perfectly calibrated model, these curves should be flat (constant entropy rate). Instead, they show systematic upward drift.
