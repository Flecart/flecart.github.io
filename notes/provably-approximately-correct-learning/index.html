PAC Learning is one of the most famous theories in learning theory. Learning theory concerns in answering questions like:

  • What is learnable? Somewhat akin to La macchina di Turing for computability theory.
  • How well can you learn something? PAC is a framework that allows to formally answer these questions. Now there is also a bayesian version of PAC in which there is a lot of research.

Some definitions#

Empirical Risk Minimizer and Errors#

The empirical risk minimizer is defined as

argc^∈Hmin​R^n​(c^)

Where the inside is the empirical error.

The Generalization error is R(c^)=Px∼D​(c^(x)=c(x))=Ex∼D​1c^(x)=c(x)​ where c^ is the learned concept (classifier) and c is the true error and x is the instance space.

Empirical error give a certain dataset is just the accuracy:

R^n​(c^)=n1​i=1∑n​1c^(x)=c(x)​

It is true that in expectation the empirical error is the same as the generalization error.

It is quite easy to see that Es∼Dm​[R^S​(c^)]=R(c^) This will be important when we will prove some bounds for the inconsistent learner case, see #The Inconsistent Case.

Concepts and Hypothesis#

We define a instance space X that is the space of the object of interest of our learning task. We define a concept a subset of X (this is why sometimes it is useful to use an indicator function to get a specific concept). A concept class is just a set of concepts. A Hypothesis class is the class of concepts that we are using to learn the true concept class.

Learning Algorithm#

Given an Hypothesis Class H and a concept c a learning algorithm in this context is a computable function that takes in input a labelled dataset (x1​,y1​),…,(xn​,yn​) and outputs an hypothesis c^∈H. One result that we will have is that we cannot have infinite precision (learn the concept class infinitely well).

PAC Learnability#

In words, we say that something is PAC learnable when given a sufficiently large sample, we can learn a concept with high probability. Formally, we say that a concept class C is PAC learnable if there exists a learning algorithm such that:

  • for all ϵ,δ>0 (respectively error parameter and confidence value)
  • for all distributions D over X,
  • if the learning algorithm is given a sample of size m≥poly(1/ϵ,1/δ,size(x))
  • => with probability at least 1−δ the algorithm outputs a hypothesis c^ such that P(c^(x)=c(x))≤ϵ. So recalling that the above is exactly the definition of risk:
PZ∼Dn​(R(c^)>ϵ)≤δ

The PAC framework is a distribution-free model : no particular assumption is made about the distribution D from which examples are drawn.

General PAC Learnability#

Above we assumed hypothesis where consistent, meaning we could correctly learn all the mappings. This is often not the case as we will probably observe noise in the observations:

PZ∼Dn​(R(c^)−c∈Cinf​R(c)>ϵ)≤δ

Efficiently Learnable#

If the sample size just depends on ε and δ then it is called efficiently learnable.

And one thing we would need to note is that sometimes the true concept is not in the hypothesis, so it's impossible to attain a risk

Example: Axis-aligned rectangles#

Suppose we are in R2 and the concept class is the set of all axis-aligned rectangles. We generate some points with these rectangles. Let our learning algorithm return the minimum rectangle that contains all the points that return true. With this learning algorithm we do not have false positives.

Now let's fix ε and δ and consider some rectangles on each side, let's make them at most equal to 4ε​. We have now that the probability of error is:

PZ∼Dn​(R(c^)>ε)≤i=1∑4​PZ∼Dn​(Z∩ri​=∅)≤4(1−ε/4)m≤4exp(−mε/4)

This implies that the number of samples needed is:

4exp(−mε/4)≤δ⇒m≥ε4​log(δ4​)

Often the sample efficiency bounds are derived in this manner in the framework of this theory.

This is a nice example on how to prove PAC related stuff. You should definitely remember the root idea of this example and how to write such proofs. There are nice exercises in the book to see if you are able to solve a PAC proof, refer to (Mohri et al. 2012).

Learning Bounds#

Finite Hypothesis Class#

If we have finite hypothesis class H then it is PAC learnable. We can get a consistent hypothesis that always classifies the training set correctly (note that here we are assuming that the dataset does not contain noise!). We need

n≥ε1​(log(∣H∣)+log(δ1​))

Proof#

Let's consider the set Hε​={h∈H:R(h)>ε}. We want to bound the probability that we get a bad hypothesis for consistent learners, that is an hypothesis that can get a zero error on the training set. If we consider the probability of the set:

P(∃h∈Hε​:R^(h)=0)≤h∈Hε​∑​P(R^(h)=0)≤∣Hε​∣(1−ε)m≤∣H∣exp(−mε)

We want to bound this probability by δ, so we have:

∣H∣exp(−mε)≤δ⇒m≥ε1​(log(∣H∣)+log(δ1​))

□

In theory, this result strongly suggest that in our finite world, we can PAC-learn everything, we only need to model finite hypothesis worlds (or hypothesis that are small enough). This will is usually the difficult part.

Universal Concept Class is not PAC-learnable#

This theorem asserts that given a instance space X={0,1}∗ and the concept class of all possible subsets, then the concept class is not PAC learnable. A simple check suggests that it is probably not learnable (although we wont provide the proof here). The main observation is that the hypothesis class is ∣H∣=22n with this setting and thus our sample complexity bounds becomes:

n≥ε1​(2nlog2+log(δ1​))

Which is not polynomial in n.

Finite Classifier Sets Bound#

Now we will tolerate that our learner does not learn the training data perfectly. We leverage Hoeffding's inequality, introduced in Central Limit Theorem and Law of Large Numbers, to prove the following theorem. If we fix an hypothesis h:X→{0,1} we have

PS∼Dm​[R^S​(h)−R(h)≤−ε]≤∣C∣exp(−2mε2)

If we put them together we get:

But note that this works for a fixed h. If we use learned hypothesis c^ it is not guaranteed that the above holds as c^ is a random variable dependent on the samples drawn (i.e. it could actually be bigger)! Fingering Argument#

Suppose you have n points. Suppose you want to take a subset d of these points, and associate to each subset two possible different hypothesis, the d-dimensional hyperplanes that passes through the points (two possible labelling, this is why we set 2). This set of hypothesis has size 2(dn​). Let's consider the one that has the best empirical error among these. Call this hypothesis ϕ^​ and the value that it can achieve to be L, and ϕ∗ the best possible among all linear classifiers. What we want to show is that ϕ^​ is a good classifier.

Good classifier#

The interesting thing of this setting is that we can use finitely many hypothesis to approximate an infinite hypothesis set.

One can show that in the infinite set of all possible classifiers, there is no classifier ϕ whose empirical error is smaller than L^(ϕ^​)−nd​. This is easy to see: by choosing d points, it can disagree at a maximum of d points, leading to that empirical error.

Standard result#

One can prove that given the setting above it is true that

P(R(c^)−c∈Cinf​R(c)>ε)≤exp(2dε−n2ε2​)(2(dn​)+1)

Then learnability is obvious: binomials grow polynomial, while n decreases exponentially, so it is bounded.

And that for n>d we have:

E[R(c^)−R]≤n2​(d+1)(log(n)+2)​

Classifiers with vanishing errors#

If we have the above assumptions plus that the best classifier has risk R(c∗)=0 then one can prove that the best classifier with the same hyperplane logic as above has this for n>d and ε≤1:

P(R(c^n​)>ε)≤2(dn​)exp(−ε(n−d))

Note that if R(c∗)=0 means that in our set exists a classifier with R^(c)≤nd​ (this is the fingering assumption).

VC Dimension#

When a theory is too powerful, that theory is not falsifiable in a Popperian sense. Meaning every type of data can be explained by that theory, suggesting that the theory does not explain anything. This is why for Buhmann says about Vapnik's work in 1982 was so interesting. In his opinion, it was able to bridge philosophy of science and mathematics.

What is VC Dimension#

The Growth Function#

The growth function of a hypothesis class H is the maximum number of dichotomies that can be induced by H on a set of n points.

mH​(n)=x1​,…,xn​∈Xmax​∣{(h(x1​),…,h(xn​)):h∈H}∣

Intuitively, the growth function is the maximum number of ways that the hypothesis class can split the data, this is later used to define the VC dimension. For example, an hypothesis class with a single hypothesis that just returns a constant, has growth function constant equal to 1.

Shattering#

We say that a set of point S of m≥1 is shattered by some hypothesis class H if the growth function is 2m. That is, the hypothesis class can split the data in all possible ways.

Intuitively, we say that an hypothesis class shatters a set of point m if whatever assignment of y we have to those point, there exists a h∈H that learns it.

Definition of VC Dimension#

The VC dimension is kinda the opposite of the Shattering number: if in the shattering we keep the set constant and try to see what hypothesis class shatters it, here we set the hypothesis class fixed, and ask The VC dimension of a hypothesis class H is the cardinality of the largest set shattered by H. This can be seen as some sort of opposite of the growth function. While the growth function focused on the number of data points, keeping H fixed, the VC dimension focuses on the number of points that H can shatter.

VC(H)=max{m:mH​(m)=2m}
